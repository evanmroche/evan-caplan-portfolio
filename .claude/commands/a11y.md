---
description: Audit a page or component for accessibility issues
---

Run an accessibility audit.

Arguments: $ARGUMENTS — a file path, route, or component name. If empty, audit the whole `app/` and `components/` tree at a high level.

Check for:

1. **Semantic HTML** — headings in order, landmarks (`<main>`, `<nav>`, `<header>`, `<footer>`), buttons vs links used correctly.
2. **Keyboard access** — every interactive element is reachable and operable without a mouse. Focus states visible against the dark theme.
3. **Images & media** — `alt` text (or explicit `alt=""` for decorative), `next/image` used where appropriate.
4. **Forms** — labels associated with inputs, error messages announced.
5. **Motion** — any animation respects `prefers-reduced-motion`. This portfolio is "cinematic" so this matters more than usual.
6. **Color & contrast** — verify text meets WCAG AA against the dark backgrounds (4.5:1 for body, 3:1 for large text).
7. **ARIA** — only used when semantic HTML can't express the intent. Flag any ARIA that duplicates or contradicts native semantics.

Report findings as a punch list with file:line citations, grouped **Blocker / Should-fix / Nit**. Don't make fixes — just report.
